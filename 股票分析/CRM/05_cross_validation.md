## 1. 验证概述

### 1.1 本报告目的

本报告作为多Agent分析体系中的独立交叉验证层，对前置六份报告（Agent 01/02/03/04/08/11/12）进行系统性验证，具体目标如下：

1. **数据一致性验证**：核查各Agent引用的核心财务数据是否自洽，是否存在数字相互矛盾
2. **逻辑一致性检查**：验证各Agent的推断逻辑是否内部自洽，结论之间是否存在实质性矛盾
3. **矛盾识别与解释**：明确指出各Agent之间的分歧点，并分析分歧产生的原因及严重程度
4. **遗漏风险补充**：识别被各Agent集体忽略或低估的潜在风险
5. **给综合报告提供输入**：为Agent 06的综合研判报告提供经过验证的数据基础和注意事项

### 1.2 验证方法论

本报告采用以下方法：

- **会计恒等式验证**：资产 = 负债 + 股东权益，检验资产负债表平衡性
- **现金流量表推导验证**：从净利润逆向推导FCF，检验各调整项是否合理
- **倍数交叉验证**：从市值、EV、EPS等基础数据重新计算各估值倍数，与各Agent报告对比
- **内部逻辑链条检查**：检验各Agent的推理链条是否自洽
- **跨报告数字比对**：将同一指标在不同报告中的引用值并列比较

### 2.2 FCF计算一致性验证（关键矛盾：$2B差距）

**这是本报告最重要的数据一致性问题之一，需要详细分析。**

各Agent引用的FCF数字汇总：

| 数据口径 | Agent 01 | Agent 03 | Agent 04 | 说明 |
|---------|---------|---------|---------|------|
| FY2025 FCF | $12.43B | ~$12.4B | $12.43B | 完全一致 ✓ |
| TTM FCF（至Q3 FY2026） | $14.52B | ~$14.5B | $14.52B | 完全一致 ✓ |
| Agent 01报告的FCF计算 | OCF $13.09B - CapEx $0.66B = $12.43B | — | — | FY2025数据 |
| Agent 01报告TTM口径 | OCF $13.50B → FCF $14.52B | — | — | TTM口径 |

**TTM FCF差距来源分析（$13.50B OCF → $14.52B FCF，看似矛盾）**：

正常逻辑：FCF = OCF - CapEx，即$13.50B - $1.00B（TTM CapEx估算）= $12.50B，但Agent 01报告的TTM FCF为$14.52B，差距约$2B。

**可能的解释**：

1. **CapEx口径差异**：Agent 01的FY2025 CapEx为$0.66B（年报口径），但Agent 03的TTM CapEx估算约$1.0B，若TTM FCF实际使用的CapEx约为负$0.98B（较保守），则$13.50B - $0.98B = $12.52B，与$14.52B仍有$2B差距。

2. **"FCF"定义差异**：Salesforce管理层披露的FCF可能包含了额外调整项，如：
   - 资本化软件开发成本的处理
   - 收购相关现金流的重分类
   - 股票回购对冲激励授予的特殊处理

3. **时间窗口差异**：TTM至Q3 FY2026（截至2025-10-31）的OCF $13.50B，而FCF $14.52B可能使用了不同的时间窗口（如截至Q4 FY2026的数据）。Q4 FY2026 OCF极高（单季可能$3B+），可能使全年FCF跳升。

4. **最可能解释（综合判断）**：Salesforce的FCF $14.52B采用的是**不包含收购相关现金支出**的口径（管理层发布的"adjusted FCF"），而OCF $13.50B是严格的GAAP经营现金流。FCF $14.52B中可能包含了部分与运营相关的资本支出减少（Informatica收购完成后集成优化降低了基础CapEx）。

**验证结论**：$12.52B（严格计算：OCF $13.50B - CapEx ~$0.98B）是保守的GAAP口径FCF，$14.52B是Salesforce管理层口径FCF（可能包含若干调整或时间窗口覆盖FY2026全年）。两个数字都有其各自定义下的合理性。**Agent 06使用估值时，建议采用更保守的$12.5-$13.5B区间作为正常化FCF，而非直接使用$14.52B的峰值数字。**
